# Basics of HTTP/HTTPS

## 1. Differentiating HTTP and HTTPS
- **HTTP**: Transfers data in plain text, making it insecure.
- **HTTPS**: Uses SSL/TLS encryption to secure data from eavesdropping.

## 2. HTTP Request and Response Structure
- **Request**: Includes Method, Path, Headers, and Body.
- **Response**: Includes Status Code, Headers, and Body.

## 3. Common HTTP Methods
- **GET**: Retrieves data.
- **POST**: Submits data.
- **PUT**: Updates a resource.
- **DELETE**: Removes a resource.

## 4. Common Status Codes
- **200 OK**: Request succeeded.
- **404 Not Found**: Resource not available.
- **500 Internal Server Error**: Server issue.
